Factory Inventory Management System

A full-stack demo for tracking inventory, orders, demand, backlog, and spending across factory warehouses. Vue 3 single-page app on the front, FastAPI service on the back, JSON files loaded into memory as the data store.

System architecture

Browser talks to Vite's dev server for the Vue app; the Vue app talks to FastAPI over HTTP for data.

Data flow

How a single filtered request — say, "orders in Q1-2025 at the West warehouse" — moves through the system.

1
User changes a filter in the UI
A view (e.g. views/Orders.vue) reads shared filter refs from the useFilters composable. Changing any filter triggers a reload function.
2
api.js builds the request
client/src/api.js assembles a URLSearchParams, skipping any filter whose value is 'all', and fires an axios GET to http://localhost:8001/api/....
3
FastAPI dispatches to an endpoint
Routes in server/main.py declare optional query params (warehouse, category, status, month) and delegate to helpers.
4
Filters are applied in Python
apply_filters() handles warehouse / category / status (category is case-insensitive). filter_by_month() expands a quarter key like Q1-2025 via QUARTER_MAP, or does a substring match on the month.
5
Pydantic validates the response
Each endpoint declares a response_model (e.g. List[Order]), so the dict shape coming out of mock_data.py is type-checked before serialization.
6
Vue stores raw data, computes the rest
Responses land in refs like allOrders or inventoryItems. Derived values (totals, groupings, chart series) live in computed properties so they recache only when inputs change.
